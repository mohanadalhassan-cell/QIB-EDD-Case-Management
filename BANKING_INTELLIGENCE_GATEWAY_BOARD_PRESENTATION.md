## COMMON BOARD QUESTIONS & ANSWERS

### Q: Is this for a specific bank?
**A:** *"This is a framework platform built for tier-1 banking institutions. The pre-authentication is completely neutral. Post-OTP, the platform reveals 'EDD Case Management' powered by Parachute Digital Intelligence. The architecture is designed for rapid institutional customization."*

### Q: How many users can this support?
**A:** *"The platform is built with enterprise scalability in mind. The frontend interface you're seeing can support thousands of concurrent users. Backend infrastructure would be determined at implementation based on institutional scale — from 100 users for regional banks to 10,000+ for global institutions."*

### Q: What about mobile access?
**A:** *"The interface is fully responsive, supporting desktop, tablet, and mobile devices. The login flow adapts gracefully from 1920px widescreen down to 480px mobile screens. All compliance references and audit features remain accessible across all devices."*

### Q: Is this GDPR/CCPA compliant?
**A:** *"Yes. The platform enforces encryption (AES-256), TLS 1.3 transport security, session logging for audit trails, and role-based access control. Data export functions are audit-ready. The frontend is WCAG 2.1 AA+ accessible. Backend API would implement GDPR-specific data handling, retention policies, and user deletion workflows."*

### Q: Can we customize the branding?
**A:** *"Absolutely. The color scheme is CSS-configurable. The compliance frameworks can be customized to match your institution's priorities. The stakeholder alignment cards can integrate with your organizational structure. The T24 field pool can be extended with local banking fields. The architecture is designed as a customizable framework."*

### Q: What about offline access?
**A:** *"This build is browser-based and requires internet connectivity for authentication. However, the frontend UI is lightweight (~250KB) and could be deployed in low-bandwidth environments. Post-authentication data could be cached locally with proper security controls for offline access scenarios."*

### Q: How does this differ from standard SSO platforms?
**A:** *"Standard SSO handles identity verification. This platform goes further by combining identity verification with compliance governance, T24 core banking integration, audit-ready export, and stakeholder alignment. It's designed specifically for financial crime investigation and due diligence workflows, not general IT access management."*
